Tokens per watt: I capped my GPU to 70% power and barely lost speed

Decoding is memory-bandwidth-bound, not compute-bound. That means you can often slash the GPU's power limit and barely lose a token per second — while running cooler, quieter, and more efficiently.

The setup

One model (Qwen2.5-Coder-7B-Instruct Q4_K_M), one CUDA GPU, the shared 12-prompt suite. We query the GPU's default power limit via nvidia-smi, then sweep down through 100 → 90 → 80 → 70 → 60 → 50% of that limit. At each step we set the cap, launch the server fresh, run the full prompt suite measuring generation tok/s, and sample the actual power draw in a background thread throughout — then compute tokens per watt.

The intuition: at small batch sizes (which is every local-LLM run), the GPU sits mostly idle waiting for VRAM reads. The arithmetic flops are done long before the next weight tile arrives. Cutting power mostly throttles compute clocks, which aren't the bottleneck — so speed barely budges while efficiency climbs.

Reproduce

Requires an NVIDIA GPU and llama-server. Running this modifies GPU power limits and requires administrator/elevated privileges (Windows: Administrator prompt; Linux: sudo). The script always restores the default limit in a finally block.

python scripts/bench.py --gpu 0
python scripts/aggregate.py && python scripts/inject.py

Set SPECBENCH_MODELS_DIR to your models folder, SPECBENCH_CUDA_EXE to your llama-server binary, and optionally SPECBENCH_MODEL_BASE (default Qwen2.5-Coder-7B-Instruct). The script expects <MODEL_BASE>-Q4_K_M.gguf.

Full guide + how to submit your hardware: CONTRIBUTING.md.

Methodology. Fixed model (Qwen2.5-Coder-7B-Instruct Q4_K_M), greedy (temp 0), 256 tokens, shared 12-prompt suite. Power draw sampled via nvidia-smi --query-gpu=power.draw every 0.5 s in a background thread throughout each suite run; average taken across all samples. tok/s/W = mean tok/s / mean power draw. VRAM is already resident — power limit changes affect clocks, not memory. Numbers vary with GPU model, driver, and thermal state.
