● Model Size: Qwen2.5-Coder 0.5B → 14B

How small a model can actually solve problems?

Sweeping Qwen2.5-Coder from 0.5B to 14B. Tiny models are fast and cheap — but somewhere down there, they stop getting things right. We measure tok/s and pass-rate at each size to find the knee: the smallest model that's actually reliable.

The setup

Five Qwen2.5-Coder-Instruct models ranging from 0.5B to 14B parameters, on the same hardware with the same prompts. Per model we record generation tok/s (shared 12-prompt suite), VRAM used, file size — plus a correctness pass-rate: a graded 10-question math/reasoning set with known integer answers. The sweet spot is where pass-rate climbs and tok/s is still acceptable.

A note on "code correctness": executing untrusted model-generated code is unsafe in a benchmark harness, so correctness here is approximated by a graded math and reasoning set (exact-answer matching). A real code-execution sandbox is a future addition.

Reproduce

Download the Qwen2.5-Coder-Instruct GGUF files into your models dir (SPECBENCH_MODELS_DIR), then:

python scripts/bench.py --backend cuda --gpu 0   # missing models are skipped
python scripts/aggregate.py && python scripts/inject.py

Full guide + how to submit your hardware: CONTRIBUTING.md.

Methodology. Greedy (temp 0), 256 tokens, shared 12-prompt suite for tok/s; a 10-question graded math/reasoning set for pass-rate. VRAM = nvidia-smi used-memory delta on model load (approximate; run on an otherwise-idle GPU). Numbers vary with GPU, llama.cpp version, and quantization.
