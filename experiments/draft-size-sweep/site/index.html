InventiveHQ · local-llm-benchmarks · experiment #10
● Speculative Decoding · Draft Size Sweep

Bigger draft model = faster? Not always.

A larger draft accepts more tokens per round — but costs more per draft step. We sweep 0.5B → 3B draft models against a fixed 14B target and measure actual speedup, VRAM cost, and acceptance rate.

The setup

Speculative decoding uses a cheap "draft" model to propose tokens that the full target model verifies in one batched pass. In experiment #1 we showed that draft overhead has to be amortized: the speedup only shows up when the acceptance rate is high enough to justify the extra draft cost. This experiment extends that thesis by asking — how much does draft model size matter?

The target is fixed: Qwen2.5-Coder-14B-Instruct-Q4_K_M, CUDA, GPU 0. We first measure a plain baseline (no speculation), then sweep three draft sizes: 0.5B (Q8_0), 1.5B (Q4_K_M), and 3B (Q4_K_M). All drafts use --spec-type draft-simple --spec-draft-n-max 5 -ngld 999 — 5 tokens drafted per round, draft fully on GPU. Speedup is draft tok/s divided by baseline tok/s; above 1.0× means faster than no speculation.

The prediction: a bigger draft accepts more tokens per round (higher acceptance rate) but also burns more VRAM and adds more latency per draft step. The sweet spot likely sits somewhere in the middle.

Reproduce

Download the target and any draft GGUFs into your models dir (SPECBENCH_MODELS_DIR), then:

python scripts/bench.py --gpu 0        # missing draft models are skipped
python scripts/aggregate.py && python scripts/inject.py

Full guide + how to submit your hardware: CONTRIBUTING.md.

Methodology. Target: Qwen2.5-Coder-14B-Instruct-Q4_K_M, CUDA backend, greedy (temp 0), 256 tokens, shared 12-prompt suite. Baseline: target only, no draft. Each draft config: draft-simple, 5 draft tokens per step, draft fully on GPU (-ngld 999). Speedup = draft tok/s / baseline tok/s. VRAM = nvidia-smi used-memory delta on model load (approximate; run on an otherwise-idle GPU). Acceptance rate parsed best-effort from server log. Sequel to experiment #1. Numbers vary with model, GPU, and llama.cpp version.
